# Telco Retention Experiment Design

## Описание проекта

Проект посвящён анализу оттока клиентов телеком-компании и проектированию retention-эксперимента. В работе объединены EDA, churn-модель, сегментация клиентов, статистическое мышление и описание A/B-теста.

Главный акцент сделан на том, как аналитик переводит результаты churn-анализа в конкретный план действий для бизнеса.

---

## Бизнес-задача

Телеком-компания хочет снизить отток клиентов. Для этого нужно понять, какие клиенты чаще уходят, какие факторы связаны с churn и какие retention-механики можно проверить экспериментально.

Практический сценарий:

    Клиенты сегментируются по риску churn → для high-risk группы запускается retention-механика → эффект проверяется через A/B-тест

---

## Данные

Используется датасет **Telco Customer Churn**.

Основные признаки:

- демография клиента
- тип контракта
- срок жизни клиента
- ежемесячные платежи
- подключённые услуги
- способ оплаты
- факт оттока

---

## Постановка задачи

Задача состоит из двух частей:

1. Аналитическая часть — найти факторы, связанные с оттоком.
2. Экспериментальная часть — описать A/B-тест для проверки retention-гипотезы.

Целевая переменная:

```text
Churn = Yes, если клиент ушёл
Churn = No, если клиент остался
```

---

## Этапы проекта

### 1. EDA

Проводится анализ:

- общего churn rate
- распределения клиентов по типам контрактов
- влияния tenure на отток
- связи ежемесячного платежа и churn
- различий между клиентами с дополнительными услугами и без них

### 2. Проверка гипотез

Формулируются и проверяются гипотезы:

- клиенты с month-to-month контрактом уходят чаще
- новые клиенты имеют более высокий риск churn
- высокий monthly charge связан с повышенным риском ухода
- дополнительные услуги могут снижать вероятность churn

### 3. Модель риска

Строится модель, которая оценивает вероятность оттока клиента. Результат используется не как финальная production-модель, а как инструмент сегментации.

### 4. Сегментация клиентов

Клиенты делятся на группы риска:

| Сегмент | Описание | Действие |
|---|---|---|
| Low risk | низкая вероятность churn | стандартная коммуникация |
| Medium risk | средний риск | персональное предложение |
| High risk | высокий риск | retention-эксперимент |

### 5. Дизайн A/B-теста

Описывается эксперимент:

- целевая группа: high-risk клиенты
- контрольная группа: стандартная коммуникация
- тестовая группа: персональное retention-предложение
- основная метрика: churn rate
- дополнительные метрики: ARPU, retention rate, uplift

---

## Пример гипотезы эксперимента

```text
Если клиентам с высоким риском оттока предложить персональный бонус или улучшенный тариф, то churn rate в тестовой группе снизится относительно контрольной группы.
```

---

## Метрики

| Метрика | Смысл |
|---|---|
| Churn Rate | доля ушедших клиентов |
| Retention Rate | доля оставшихся клиентов |
| Uplift | изменение метрики в тестовой группе |
| ARPU | средняя выручка на клиента |
| ROC-AUC | качество ранжирования клиентов по риску |

---

## Ключевые выводы

- клиенты с коротким сроком жизни обычно имеют более высокий риск оттока
- тип контракта может быть одним из сильных факторов churn
- retention-кампанию лучше запускать не на всех клиентах, а на сегментах риска
- A/B-тест помогает проверить, снижает ли механика churn, а не просто выглядит логичной

---

## Бизнес-ценность

Проект показывает, как использовать churn-аналитику для практического retention-процесса:

- выявление клиентов с высоким риском
- приоритизация retention-бюджета
- проверка гипотез через эксперимент
- снижение оттока и сохранение выручки

---

## Ограничения проекта

- датасет учебный и не содержит истории коммуникаций
- нет информации о стоимости retention-предложения
- нет реальных результатов эксперимента
- модель требует проверки на новых данных
