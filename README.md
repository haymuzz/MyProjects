## Ключевые проекты

### 1. Olist Marketplace Product Analytics

**Цель:** проанализировать работу e-commerce маркетплейса и найти факторы, влияющие на клиентский опыт.

Что сделано:

- рассчитаны основные метрики заказов и выручки;
- проведен анализ динамики продаж;
- исследованы задержки доставки;
- выявлены категории товаров с повышенным риском плохих отзывов;
- подготовлены SQL-запросы для регулярной отчетности;
- сформулированы рекомендации для бизнеса.

**Результат:** определены проблемные зоны в доставке и клиентском сервисе, которые могут влиять на повторные покупки и оценки пользователей.

---

### 2. Fraud Detection & Risk Scoring

**Цель:** построить аналитическое решение для выявления потенциально мошеннических транзакций.

Что сделано:

- проведен EDA транзакционных данных;
- изучен дисбаланс классов;
- построена baseline-модель классификации;
- рассчитаны ROC-AUC, PR-AUC, Precision, Recall;
- подобран threshold под бизнес-задачу;
- подготовлены SQL-запросы для мониторинга подозрительных операций.

**Результат:** создан риск-скор транзакций, который помогает приоритизировать проверки и снижать нагрузку на антифрод-команду.

---

### 3. Store Sales Forecasting ETL

**Цель:** построить воспроизводимый пайплайн для подготовки данных и прогнозирования продаж.

Что сделано:

- описан процесс загрузки и обработки данных;
- проведена очистка и агрегация данных;
- реализованы проверки качества данных;
- подготовлена структура ETL-пайплайна;
- построена baseline-модель прогноза продаж;
- рассчитаны MAE, RMSE и MAPE.

**Результат:** подготовлен пайплайн, который можно использовать для регулярного обновления прогноза продаж и контроля качества данных.

---

### 4. Telco Retention Experiment Design

**Цель:** проанализировать отток клиентов и подготовить дизайн эксперимента по удержанию.

Что сделано:

- рассчитан churn rate;
- выделены сегменты клиентов с высоким риском оттока;
- сформулированы гипотезы удержания;
- подготовлена логика A/B-теста;
- определены основные и guardrail-метрики;
- описаны ограничения анализа.

**Результат:** сформирован план retention-эксперимента, который можно использовать для проверки гипотез по снижению оттока.

---

### 5. MovieLens Recommendation API

**Цель:** построить простую рекомендательную систему и оформить ее в виде API.

Что сделано:

- проведен анализ пользовательских оценок;
- построена baseline-рекомендательная модель;
- подготовлена SQLite-база;
- реализован API на FastAPI;
- описаны сценарии использования;
- рассчитаны базовые метрики качества рекомендаций.

**Результат:** создан прототип рекомендательного сервиса, который может возвращать персональные рекомендации пользователям.

---

### 6. Customer Lifetime Value Segmentation

**Цель:** сегментировать клиентов и определить группы с разной ценностью для бизнеса.

Что сделано:

- рассчитаны Recency, Frequency, Monetary;
- выделены RFM-сегменты;
- проведена кластеризация клиентов;
- рассчитаны показатели клиентской ценности;
- подготовлены рекомендации для CRM-кампаний;
- написаны SQL-запросы для клиентской витрины.

**Результат:** сформированы клиентские сегменты для персонализированных коммуникаций, удержания и роста повторных покупок.
