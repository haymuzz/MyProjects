# MovieLens Recommendation API

## Описание проекта

Проект представляет собой учебный сервис рекомендаций фильмов. Пользователь получает список фильмов, которые могут быть ему интересны на основе истории оценок и похожести пользовательского поведения.

Проект объединяет аналитику данных, базовую рекомендательную систему, работу с базой данных и REST API.

---

## Цель проекта

Показать базовую реализацию рекомендательной системы и продемонстрировать навыки работы с данными, API, SQLite, Python и алгоритмами рекомендаций.

---

## Основная функциональность

- загрузка фильмов и пользовательских оценок
- построение матрицы пользователь-фильм
- расчёт похожести пользователей или фильмов
- формирование top-K рекомендаций
- исключение фильмов, которые пользователь уже оценил
- хранение подготовленных данных в SQLite
- выдача рекомендаций через REST API

---

## Данные

Используется датасет **MovieLens 20M Dataset**.

Основные таблицы:

- `ratings.csv` — оценки пользователей
- `movies.csv` — справочник фильмов
- `tags.csv` — пользовательские теги
- `links.csv` — внешние идентификаторы фильмов

---

## Структура базы данных

В проекте можно использовать три основные сущности.

### Users

Пользователи рекомендательной системы.

Основные поля:

- `user_id`

### Movies

Справочник фильмов.

Основные поля:

- `movie_id`
- `title`
- `genres`

### Ratings

Оценки фильмов пользователями.

Основные поля:

- `user_id`
- `movie_id`
- `rating`
- `timestamp`

Связи:

- один пользователь может поставить много оценок
- один фильм может иметь много оценок
- таблица `Ratings` связывает пользователей и фильмы

---

## Алгоритм рекомендаций

В проекте реализуется baseline-подход к рекомендациям.

Логика работы:

1. Из данных загружаются оценки пользователей.
2. Строится матрица пользователь-фильм.
3. Пропущенные значения обрабатываются для расчёта похожести.
4. Рассчитывается similarity между пользователями или фильмами.
5. Для выбранного пользователя выбираются фильмы с максимальным рекомендательным score.
6. Из рекомендаций исключаются уже оценённые фильмы.
7. Пользователю возвращается top-K список.

## Метрики качества

Для оценки рекомендательной системы можно использовать:

- Precision@K
- Recall@K
- MAP@K
- Hit Rate
- Coverage

Top-K метрики лучше отражают пользовательский сценарий, чем обычная accuracy.

---

## Пример пользовательского сценария

1. Пользователь открывает сервис.
2. Система находит его историю оценок.
3. Алгоритм рассчитывает похожие фильмы или похожих пользователей.
4. Уже просмотренные фильмы исключаются.
5. Пользователь получает список рекомендаций.

## Ограничения проекта

- baseline-модель не решает полностью проблему cold-start
- рекомендации строятся на исторических оценках
- качество зависит от плотности пользовательских оценок
- для production-сценария нужен мониторинг качества рекомендаций

## Установка и запуск

Установить зависимости:

```bash
pip install -r requirements.txt
```

Скачать данные:

```bash
kaggle datasets download -d grouplens/movielens-20m-dataset
unzip movielens-20m-dataset.zip -d data/raw
```

Запустить ноутбук:

```bash
jupyter notebook notebooks/movielens_recommendation_api.ipynb
```

Запустить API:

```bash
uvicorn src.main:app --reload
```

После запуска API будет доступен по адресу:

```text
http://127.0.0.1:8000
```

Документация FastAPI:

```text
http://127.0.0.1:8000/docs
```

## Итог

Проект демонстрирует базовый сервис рекомендаций: подготовка данных, расчёт рекомендаций, хранение информации в SQLite и выдача результата через REST API.

## Полный исполняемый код

В проект добавлен файл:

```bash
python src/full_analysis.py
```

Он выполняет весь цикл анализа:

- ищет CSV-файлы Kaggle в `data/raw`;
- если данных нет, создаёт демонстрационный датасет, чтобы код запускался сразу;
- проводит проверку качества данных;
- строит EDA и бизнес-метрики;
- сохраняет графики в `reports/figures`;
- сохраняет итоговые выводы в `reports/summary.md`.

Ноутбук в папке `notebooks/` содержит тот же полный пайплайн и подходит для просмотра на GitHub.
