# Чек-лист репозитория

Перед публикацией на GitHub проверь:

## Общий вид
- [ ] В корневом README есть краткое описание тебя и контакты.
- [ ] Есть ссылка на резюме.
- [ ] Все проекты отображены в таблице.
- [ ] У каждого проекта есть статус: planned / in progress / completed.
- [ ] Нет больших CSV-файлов в репозитории.

## Для каждого проекта
- [ ] README содержит бизнес-задачу.
- [ ] Есть ссылка на Kaggle dataset.
- [ ] Есть структура папок `data/raw`, `notebooks`, `src`, `reports/figures`.
- [ ] Есть `requirements.txt`.
- [ ] Есть итоговые метрики.
- [ ] Есть бизнес-выводы.
- [ ] Есть ограничения и next steps.

## Финальная доработка GitHub
- [ ] Папки пронумерованы в правильном порядке.
- [ ] Названия проектов читаются как реальные бизнес-кейсы.
- [ ] Графики сохранены в `reports/figures`.
- [ ] В notebook нет лишних тестовых ячеек.
- [ ] В README нет фраз "я просто попробовал" или "учебный проект" в слабой формулировке.
