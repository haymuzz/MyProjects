# Olist Marketplace Product Analytics

## Описание проекта

Проект посвящён продуктовому анализу маркетплейса на данных Olist. В работе рассматриваются заказы, платежи, доставка, отзывы клиентов, категории товаров и география покупателей.

Основной фокус сделан на том, как аналитик может найти проблемные зоны клиентского опыта и перевести результаты анализа в бизнес-рекомендации.

---

## Бизнес-задача

Маркетплейсу важно понимать, какие факторы влияют на клиентский опыт и повторные покупки. Если доставка задерживается или отдельные категории часто получают низкие оценки, это напрямую влияет на удовлетворённость клиентов и будущую выручку.

Практический сценарий:

    Команда продукта хочет понять, где теряется качество сервиса → аналитик рассчитывает метрики заказов, доставки и отзывов → бизнес получает список зон для улучшения

---

## Данные

Используется датасет **Brazilian E-Commerce Public Dataset by Olist**.

Основные таблицы:

- `orders` — информация о заказах и статусах
- `order_items` — товары внутри заказов
- `order_payments` — платежи
- `order_reviews` — отзывы и оценки клиентов
- `customers` — данные покупателей
- `products` — справочник товаров
- `sellers` — продавцы
- `geolocation` — географические данные

Файлы данных не добавлены в репозиторий из-за размера и условий распространения Kaggle.

---

## Основные задачи

- загрузить и подготовить таблицы
- проверить пропуски, дубликаты и корректность типов данных
- рассчитать GMV, количество заказов, средний чек и динамику продаж
- проанализировать сроки доставки и задержки
- изучить связь между доставкой и оценками клиентов
- выделить категории с высоким риском негативных отзывов
- подготовить SQL-запросы для аналитической отчётности
- сформулировать продуктовые рекомендации

---

## Структура анализа

### 1. Подготовка данных

На этом этапе выполняется загрузка таблиц, проверка ключей, объединение заказов с товарами, платежами и отзывами. Также создаются временные признаки: месяц заказа, срок доставки, задержка относительно ожидаемой даты.

### 2. Анализ заказов и выручки

Рассчитываются базовые бизнес-метрики:

- количество заказов
- количество покупателей
- GMV
- средний чек
- выручка по месяцам
- выручка по категориям
- доля отменённых заказов

### 3. Анализ доставки

Изучаются сроки доставки и отклонение фактической даты доставки от ожидаемой.

Ключевой вопрос:

    Какие регионы, продавцы или категории чаще всего приводят к задержкам?

### 4. Анализ отзывов

Оценки клиентов связываются со сроками доставки, категорией товара и стоимостью заказа. Это позволяет определить, какие факторы чаще всего связаны с низкими оценками.

### 5. Продуктовые выводы

Финальный этап переводит результаты анализа в понятные рекомендации для бизнеса: какие категории контролировать, где улучшать SLA и какие сегменты клиентов требуют внимания.

---

## Примеры метрик

| Метрика | Смысл | Как использовать |
|---|---|---|
| GMV | общий оборот заказов | оценка масштаба продаж |
| Average Order Value | средний чек | анализ ценности заказа |
| Delivery Delay Rate | доля заказов с задержкой | контроль качества доставки |
| Review Score | оценка клиента | измерение клиентского опыта |
| Cancellation Rate | доля отмен | поиск проблем в заказах |

---

## Ключевые выводы

- доставка является одним из главных факторов клиентского опыта
- задержки могут снижать оценки даже при успешной покупке
- категории товаров отличаются по уровню риска негативных отзывов
- аналитическая витрина по заказам помогает регулярно контролировать качество сервиса
- для маркетплейса важно отслеживать не только выручку, но и операционные метрики

## Ограничения проекта

- датасет исторический и не отражает текущую ситуацию на рынке
- нет полной информации о маркетинговых кампаниях
- нет данных о повторных визитах пользователей вне заказов
- выводы требуют проверки на новых данных

## Как запустить

Установить зависимости:

```bash
pip install -r requirements.txt
```

Скачать данные через Kaggle API:

```bash
kaggle datasets download -d olistbr/brazilian-ecommerce
unzip brazilian-ecommerce.zip -d data/raw
```

Запустить ноутбук:

```bash
jupyter notebook notebooks/olist_marketplace_product_analytics.ipynb
```
