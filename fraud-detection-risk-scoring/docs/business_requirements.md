# Бизнес-требования - Fraud Detection & Risk Scoring

## Проблема

Банку или платежному сервису нужно находить мошеннические операции так, чтобы снижать финансовые потери и не блокировать слишком много нормальных клиентов.

## Пользователи

- аналитик данных;
- продуктовый менеджер;
- бизнес-заказчик;
- команда разработки/BI, если проект требует автоматизации.

## Функциональные требования

FR-1: провести EDA транзакций и сравнить fraud / non-fraud поведение.
FR-2: оценить дисбаланс классов и выбрать правильные метрики.
FR-3: построить baseline: Logistic Regression / Random Forest.
FR-4: улучшить модель через class_weight, undersampling или SMOTE.
FR-5: подобрать threshold под бизнес-стоимость ошибок.

## Нефункциональные требования

- Анализ должен быть воспроизводимым.
- Большие исходные данные не должны попадать в Git.
- README должен содержать бизнес-выводы, а не только техническое описание.
- Метрики должны быть интерпретируемы для заказчика.

## Критерии приёмки

- Проект запускается по инструкции.
- Все ключевые графики сохранены в `reports/figures`.
- В README указаны финальные метрики.
- Есть раздел с ограничениями и следующими шагами.
