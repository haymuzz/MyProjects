# Customer Lifetime Value & RFM Segmentation

## Описание проекта

Проект посвящён сегментации клиентов интернет-магазина и оценке клиентской ценности. В работе используются RFM-анализ, базовая кластеризация и SQL-запросы для подготовки клиентской витрины.

Основная цель — показать, как из транзакционных данных получить понятные CRM-сегменты и рекомендации для удержания клиентов.

---

## Бизнес-задача

Компании нужно понимать, какие клиенты приносят больше выручки, кто покупает часто, кто давно не возвращался и какие группы требуют разных маркетинговых действий.

Практический сценарий:

    История покупок → RFM-признаки → клиентские сегменты → персональные CRM-кампании

---

## Данные

Используется датасет **Online Retail Customer Clustering**.

Основные поля:

- `InvoiceNo` — номер заказа
- `StockCode` — код товара
- `Description` — описание товара
- `Quantity` — количество
- `InvoiceDate` — дата заказа
- `UnitPrice` — цена единицы товара
- `CustomerID` — идентификатор клиента
- `Country` — страна

---

## Основные задачи

- загрузить и очистить транзакционные данные
- удалить возвраты, некорректные цены и пропуски CustomerID
- рассчитать выручку по заказам
- сформировать клиентскую витрину
- рассчитать Recency, Frequency, Monetary
- выделить RFM-сегменты
- применить кластеризацию
- подготовить рекомендации для CRM

---

## RFM-логика

RFM-анализ строится на трёх признаках:

| Признак | Смысл | Интерпретация |
|---|---|---|
| Recency | сколько дней прошло с последней покупки | чем меньше, тем клиент активнее |
| Frequency | сколько заказов сделал клиент | чем больше, тем выше лояльность |
| Monetary | сколько денег принёс клиент | чем больше, тем выше ценность |

---

## Этапы проекта

### 1. Подготовка данных

На этом этапе выполняется очистка транзакций, обработка пропусков, удаление возвратов и расчёт суммы покупки.

### 2. Формирование клиентской витрины

Данные агрегируются на уровне клиента:

```text
customer_id → recency, frequency, monetary, avg_order_value, total_revenue
```

### 3. RFM-сегментация

Клиенты получают RFM-score и распределяются по сегментам:

- Champions
- Loyal Customers
- Potential Loyalists
- At Risk
- Lost Customers

### 4. Кластеризация

Для дополнительного анализа применяется кластеризация. Она помогает найти группы клиентов с похожим поведением без заранее заданных правил.

### 5. CRM-рекомендации

Для каждого сегмента формулируются действия:

| Сегмент | Возможное действие |
|---|---|
| Champions | персональные предложения и ранний доступ |
| Loyal Customers | программа лояльности |
| Potential Loyalists | стимулирование повторной покупки |
| At Risk | реактивационная кампания |
| Lost Customers | win-back предложение |

---

## Ключевые выводы

- небольшая доля клиентов может приносить значительную часть выручки
- клиенты с высокой Frequency и Monetary требуют отдельной стратегии удержания
- Recency помогает быстро находить клиентов с риском потери
- RFM-сегментация хорошо подходит для объяснимой CRM-аналитики

## Ограничения проекта

- нет данных о маркетинговых контактах
- нет информации о себестоимости и маржинальности
- часть клиентов может быть без CustomerID
- RFM не учитывает весь контекст поведения пользователя
