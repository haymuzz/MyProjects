# Store Sales Forecasting ETL Pipeline

## Описание проекта

Проект посвящён прогнозированию продаж магазинов и построению воспроизводимого ETL-пайплайна. В работе объединены аналитика временных рядов, подготовка витрин, контроль качества данных и базовая автоматизация через Airflow.

Главная идея проекта — показать, как аналитик может не только построить прогноз, но и оформить процесс так, чтобы он регулярно обновлялся и был понятен команде.

---

## Бизнес-задача

Ритейлу нужно прогнозировать продажи, чтобы лучше планировать запасы, промо-акции и операционную нагрузку магазинов. Ошибка прогноза может приводить либо к дефициту товаров, либо к излишним остаткам.

Практический сценарий:

    Новые продажи загружаются каждый день → ETL обновляет витрину → модель строит прогноз → бизнес получает данные для планирования

---

## Данные

Используется датасет **Store Sales - Time Series Forecasting**.

Основные таблицы:

- `train.csv` — исторические продажи
- `test.csv` — период для прогноза
- `stores.csv` — информация о магазинах
- `oil.csv` — цены на нефть
- `holidays_events.csv` — праздники и события
- `transactions.csv` — количество транзакций

## Этапы проекта

### 1. Загрузка данных

На этом этапе данные скачиваются с Kaggle и помещаются в папку `data/raw`. Далее выполняется проверка структуры файлов и базовая валидация.

### 2. Data Quality

Проверяются:

- пропуски
- дубликаты
- корректность дат
- отрицательные продажи
- полнота временного ряда
- соответствие магазинов и товарных групп справочникам

### 3. Feature Engineering

Создаются признаки:

- день недели
- месяц
- лаги продаж
- скользящие средние
- наличие промо
- праздничные дни
- агрегаты по магазинам и категориям

### 4. Витрина данных

Формируется аналитическая витрина на уровне:

```text
date + store_nbr + family
```

Витрина может использоваться для отчётности и обучения модели.

### 5. Forecasting

Строится baseline-прогноз и ML-модель. Для оценки качества используются:

- MAE
- RMSE
- MAPE

### 6. Airflow DAG

В проект добавлена структура DAG, которая показывает, как можно автоматизировать регулярное обновление данных и прогноза.

---

## Пример задач DAG

```text
extract_data → validate_data → build_features → train_model → save_forecast → generate_report
```

## Ограничения проекта

- Kaggle-данные не обновляются в реальном времени
- baseline-модель не учитывает все внешние факторы
- без production-инфраструктуры DAG является демонстрационным
- качество прогноза нужно проверять на временных срезах

## Итог

Проект показывает полный цикл работы с прогнозированием продаж: подготовка данных, проверки качества, построение витрины, baseline-модель, метрики качества и демонстрационная автоматизация через Airflow.

## Полный исполняемый код

В проект добавлен файл:

```bash
python src/full_analysis.py
```

Он выполняет весь цикл анализа:

- ищет CSV-файлы Kaggle в `data/raw`;
- если данных нет, создаёт демонстрационный датасет, чтобы код запускался сразу;
- проводит проверку качества данных;
- строит EDA и бизнес-метрики;
- сохраняет графики в `reports/figures`;
- сохраняет итоговые выводы в `reports/summary.md`.

Ноутбук в папке `notebooks/` содержит тот же полный пайплайн и подходит для просмотра на GitHub.
