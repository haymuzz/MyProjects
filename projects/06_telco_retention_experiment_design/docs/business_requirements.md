# Бизнес-требования - Telco Retention Experiment Design

## Проблема

Телеком-компании нужно понять причины оттока и проверить, какие retention-кампании могут снизить churn.

## Пользователи

- аналитик данных;
- продуктовый менеджер;
- бизнес-заказчик;
- команда разработки/BI, если проект требует автоматизации.

## Функциональные требования

FR-1: провести EDA клиентской базы и churn rate.
FR-2: рассчитать churn по контрактам, tenure, услугам и платежам.
FR-3: построить interpretable model для churn risk.
FR-4: сегментировать клиентов по риску и ценности.
FR-5: сформулировать retention гипотезы.

## Нефункциональные требования

- Анализ должен быть воспроизводимым.
- Большие исходные данные не должны попадать в Git.
- README должен содержать бизнес-выводы, а не только техническое описание.
- Метрики должны быть интерпретируемы для заказчика.

## Критерии приёмки

- Проект запускается по инструкции.
- Все ключевые графики сохранены в `reports/figures`.
- В README указаны финальные метрики.
- Есть раздел с ограничениями и следующими шагами.
